| 1 | Someone just donated a laptop to your school and now you wish to install Linux on it. There is no manual and you were forced to boot it from a USB thumb drive with no graphics whatsoever. You do get a shell terminal and you know, for every processor you have there will be a line for it in the `/proc/cpuinfo` file:<br><br>`processor : 0`<br>`vendor_id : GenuineIntel`<br>`cpu family : 6`<br>`model : 158`<br>(lines skipped)<br>`processor : 1`<br>`vendor_id : GenuineIntel`<br>`cpu family : 6`<br>`model : 158`<br>(more lines skipped)<br><br>◦ Using the commands `grep` and `wc` display how many processors you have.<br>◦ Do the same thing with `sed` instead of `grep`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      | Here are two options:<br><br>`$ cat /proc/cpuinfo \| grep processor \| wc -l`<br>`$ grep processor /proc/cpuinfo \| wc -l`<br><br>Now that you know there are several ways you can do the same thing, when should you be using one or the other? It really depends on several factors, the two most important ones being performance and readability. Most of the time you will use shell commands inside shell scripts to automate your tasks and the larger and more complex your scripts become the more you need to worry about keeping them fast.<br><br>Now, instead of `grep` we will try this with `sed`:<br><br>`$ sed -n /processor/p /proc/cpuinfo \| wc -l`<br><br>Here we used `sed` with the `-n` parameter so `sed` will not print anything except for what matches with the expression `processor`, as instructed by the `p` command. As we did in the `grep` solutions, `wc -l` will count the number of lines, thus the number of processors we have.<br><br>Study this next example:<br><br>`$ sed -n /processor/p /proc/cpuinfo \| sed -n '$='`<br><br>This command sequence provides identical results to the previous example where the output of `sed` was piped into a `wc` command. The difference here is that instead of using `wc -l` to count the number of lines, `sed` is again invoked to provide equivalent functionality. Once more, we are suppressing the output of `sed` with with the `-n` option, except for the expression that we are explicitly calling, which is `'$='`. This expression tells `sed` to match the last line (`$`) and then to print that line number (`=`).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| 2 | Explore your local `/etc/passwd` file with the `grep`, `sed`, `head` and `tail` commands per the tasks below:<br><br>◦ Which users have access to a Bash shell?<br>◦ Your system has various users that exists to handle specific programs or for administrative purposes. They do not have access to a shell. How many of those exist in your system?<br>◦ How many users and groups exist in your system (remember: use only the `/etc/passwd` file)?<br>◦ List only the first line, the last line and the tenth line of your `/etc/passwd` file.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                | **Which users have access to a Bash shell?**<br><br>`$ grep ":/bin/bash$" /etc/passwd`<br><br>We will improve this answer by only displaying the name of the user that utilizes the Bash shell.<br><br>`$ grep ":/bin/bash$" /etc/passwd \| cut -d: -f1`<br><br>The user name is the first field (`-f1` parameter of the `cut` command) and the `/etc/passwd` file uses `:` as separators (`-d:` parameter of the `cut` command) we just pipe the output of the `grep` command to the appropriate `cut` command.<br><br>**Your system has various users that exists to handle specific programs or for administrative purposes. They do not have access to a shell. How many of those exist in your system?**<br><br>The easiest way to find this is by printing out the lines for accounts that do not use the Bash shell:<br><br>`$ grep -v ":/bin/bash$" /etc/passwd \| wc -l`<br><br>**How many users and groups exist in your system (remember: use only the `/etc/passwd` file)?**<br><br>The first field of any given line in your `/etc/passwd` file is the user name, the second is typically an `x` indicating the user password is not stored here (it is encrypted in the `/etc/shadow` file). The third is the user id (`UID`) and the fourth is the group id (`GID`). So this should give us the number of users:<br><br>`$ cut -d: -f3 /etc/passwd \| wc -l`<br><br>Well, most of the time it will. However, there are situations where you will set different super users or other special kinds of users sharing the same `UID` (user id). So, to be on the safe side we will pipe the result of our `cut` command to the `sort` command and then count the number of lines.<br><br>`$ cut -d: -f3 /etc/passwd \| sort -u \| wc -l`<br><br>Now, for the number of groups:<br><br>`$ cut -d: -f4 /etc/passwd \| sort -u \| wc -l`<br><br>**List only the first line, the last line and the tenth line of your `/etc/passwd` file**<br><br>This will do:<br><br>`$ sed -n -e '1'p -e '10'p -e '$'p /etc/passwd`<br><br>Remember that the parameter `-n` tells `sed` not to print anything other than what is specified by the `p` command. The dollar sign (`$`) used here is a regular expression meaning the last line of the file. |

You might be tempted to try this:<br><br>`$ sed -n /1000/p mypasswd`<br><br>In this case you will also get this line:<br><br>`carol:x:1000:2000:Carol Smith,Finance,,,Main Office:/home/carol:/bin/bash`<br><br>You know this is not correct since Carol Smith is a member of `GID 2000` and the match occurred because of the `UID`. However, you may have noticed that after the `GID` the next field starts with an upper case character. We can use a regular expression to solve this problem.<br><br>`$ sed -n /:1000:[A-Z]/p mypasswd`<br><br>The expression `[A-Z]` will match any single upper case character. You will learn more about this in the respective lesson.<br><br>**List only the full names of all the users for this group (use `sed` and `cut`):**<br><br>Use the same technique you used to solve the first part of this exercise and pipe it to a `cut` command.<br><br>`$ sed -n /:1000:[A-Z]/p mypasswd \| cut -d: -f5`<br><br>`Dave Edwards,Finance,,,Main Office`<br>`Emma Jones,Finance,,,Main Office`<br>`Frank Cassidy,Finance,,,Main Office`<br>`Grace Kearns,Engineering,,,Main Office`<br>`Henry Adams,Sales,,,Main Office`<br>`John Chapel,Sales,,,Main Office`<br><br>Not quite there! Do note how the fields inside your results can be separated by `,`. So we will pipe the output to another `cut` command, using the `,` as a delimiter.<br><br>`$ sed -n /:1000:[A-Z]/p mypasswd \| cut -d: -f5 \| cut -d, -f1`<br><br>`Dave Edwards`<br>`Emma Jones`<br>`Frank Cassidy`<br>`Grace Kearns`<br>`Henry Adams`<br>`John Chapel`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
